# Agent 测试交互流程规范

## 总体流程

```
变更检测 → Agent 分析 → 提议测试计划 → 用户确认 → 生成测试用例 → 用户确认 → 执行测试 → 汇报结果
```

## 阶段一：变更分析

**触发条件**: 扫描脚本检测到新提交并生成 report

**Agent 职责**:
1. 读取 `reports/` 下最新报告
2. 分析变更内容，写入 `summary.md`
3. 根据变更内容判断影响的测试层级

**输出**: `summary.md` 包含变更概述、影响范围、测试建议、潜在风险

## 阶段二：提议测试计划

**Agent 职责**:
1. 基于 `summary.md` 生成测试计划
2. 写入 `test-config/test-plan.md`
3. 向用户展示计划并等待确认

**测试计划格式**:

```markdown
# 测试计划 - <项目名称>

## 变更来源
- 提交范围: xxx..xxx
- 变更文件数: N

## 建议测试范围

### L1 单元测试
| 用例 | 测试目标 | 优先级 |
|------|---------|--------|
| ... | ... | P0/P1/P2 |

### L2 接口测试
| 用例 | 接口 | 方法 | 优先级 |
|------|------|------|--------|
| ... | ... | GET/POST | P0/P1/P2 |

### L3 E2E 测试
| 用例 | 业务流程 | 优先级 |
|------|---------|--------|
| ... | ... | P0/P1/P2 |

### L4 UI 测试
| 用例 | 页面/组件 | 交互 | 优先级 |
|------|----------|------|--------|
| ... | ... | ... | P0/P1/P2 |

## 环境要求
- 需要的服务/中间件
- 环境配置说明

## 预估执行时间
- L1: ~X分钟
- L2: ~X分钟
- L3: ~X分钟
- L4: ~X分钟
```

**用户交互**: 向用户展示测试计划，询问：
- 是否同意此计划？
- 是否需要调整测试范围？
- 是否需要补充特定测试场景？
- 是否需要提供环境配置（密码、Token 等）？

## 阶段三：生成测试用例

**触发条件**: 用户确认测试计划

**Agent 职责**:
1. 根据确认的计划生成具体测试代码
2. 测试代码写入 `tests/` 对应层级目录
3. 向用户展示用例概览

**用户交互**: 向用户展示生成的用例，询问：
- 用例是否覆盖了需要测试的场景？
- 是否需要修改或补充用例？

## 阶段四：执行测试

**触发条件**: 用户确认测试用例

**Agent 职责**:
1. 准备测试环境（安装依赖、启动容器）
2. 按 L1 → L2 → L3 → L4 顺序执行
3. 收集测试结果
4. 生成测试报告到 `results/latest/`

**执行过程**:
- 每个层级执行完后汇报该层结果
- 遇到 FAIL/ERROR 时记录并继续执行后续用例
- UI 测试失败时自动截图保存

## 阶段五：结果汇报

**Agent 职责**:
1. 汇总所有层级测试结果
2. 生成 `results/latest/report.md`
3. 向用户汇报

**汇报内容**:
- 总体通过率
- 各层级结果汇总
- 失败用例详情（错误日志、截图）
- 建议的后续操作

**用户决策点**:
- 测试全部通过 → 可选择提交 issue 到原项目或结束
- 测试有失败 → 选择：修复后重试 / 记录 issue / 调整用例后重试

## 阶段六：问题反馈

**触发条件**: 用户确认提交问题到原项目

**Agent 职责**:
1. 整理测试发现的问题
2. 生成 issue 内容（复现步骤、错误日志、环境信息、建议修复）
3. 提交到原项目仓库

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 测试环境无法启动 | 记录错误，报告用户，等待用户配置 |
| 测试框架未安装 | 自动安装（需用户确认），或提示用户手动安装 |
| 外部依赖不可用 | 跳过依赖该服务的用例，标记为 SKIP |
| 执行超时 | 单个用例超时 5 分钟自动终止，标记为 ERROR |

## 优先级定义

| 级别 | 说明 | 触发条件 |
|------|------|---------|
| P0 | 必须测试 | 变更涉及核心业务逻辑、安全相关、数据完整性 |
| P1 | 应该测试 | 变更涉及一般功能、API 接口、页面交互 |
| P2 | 可选测试 | 变更涉及样式调整、文档更新、配置变更 |
